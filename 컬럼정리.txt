KLOOK 크롤러 데이터 컬럼 분석 및 정리

=== 현재 컬럼 분석 ===

【필수 컬럼 (Core)】
- title: 상품명 - 크롤링의 기본이 되는 핵심 데이터
- url: 상품 페이지 URL - 중복 제거 및 재방문을 위한 고유 식별자
- price: 가격 정보 - 비즈니스 분석의 핵심 데이터

【중요 컬럼 (Important)】  
- rating: 평점 - 상품 품질 지표, 필터링/정렬 기준
- review_count: 리뷰 개수 - 인기도 및 신뢰도 지표
- location_tags: 위치 학습 시스템 추출 태그 - 지역별 분류 핵심
- description: 상품 설명 - 위치 학습 및 상세 분석 소스

【부가 컬럼 (Optional)】
- image_url: 상품 이미지 URL - 시각적 데이터, 저장 용량 고려 필요
- category: 카테고리 정보 - 분류 및 필터링용

=== 추가 고려 컬럼 ===

【메타 정보】
- crawl_date: 크롤링 수집 일시 - 데이터 신선도 추적
- city_name: 대상 도시명 - 멀티 도시 크롤링 시 필수
- page_number: 수집된 페이지 번호 - 페이지네이션 추적
- data_source: 데이터 소스 (klook) - 향후 다중 소스 확장 대비

【비즈니스 정보】
- original_price: 원가 - 할인율 계산용
- discount_rate: 할인율 - 프로모션 분석용
- availability: 예약 가능 여부 - 실시간 상태
- duration: 체험 소요시간 - 일정 계획용
- min_group_size: 최소 인원 - 예약 조건
- max_group_size: 최대 인원 - 예약 조건

【위치 정보】
- exact_location: 정확한 위치/주소 - GPS 좌표 또는 상세 주소
- nearby_landmarks: 주변 랜드마크 - 위치 학습 보조
- district: 구/지역 - 세부 지역 분류

【품질 지표】
- verified_badge: 인증 배지 여부 - 신뢰도 지표
- bestseller_badge: 베스트셀러 배지 - 인기도 지표
- cancellation_policy: 취소 정책 - 예약 조건
- instant_confirmation: 즉시 확정 여부 - 편의성 지표

=== 삭제 고려 컬럼 ===

【저장 용량 최적화】
- image_url: 이미지 URL은 용량 대비 활용도가 낮을 수 있음
  → 대안: 필요시에만 별도 수집하는 선택적 컬럼으로 변경

【중복성 제거】
- category: KLOOK 자체 카테고리보다 location_tags가 더 유용할 수 있음
  → 대안: 통합하거나 더 정확한 분류 체계 도입

=== 우선순위별 컬럼 그룹 ===

【Phase 1: 필수 최소 세트】
✓ title (필수)
✓ url (필수) 
✓ price (필수)
✓ rating (중요)
✓ review_count (중요)
✓ location_tags (중요)
+ crawl_date (추가)
+ city_name (추가)

【Phase 2: 비즈니스 확장】
+ original_price
+ discount_rate  
+ availability
+ duration
+ exact_location

【Phase 3: 고급 분석】
+ verified_badge
+ bestseller_badge
+ cancellation_policy
+ nearby_landmarks
+ min/max_group_size

=== 컬럼 설계 권장사항 ===

1. 필수 컬럼 8개로 시작 (Phase 1)
2. 크롤링 성능 확인 후 단계적 확장
3. 저장 용량과 처리 속도 모니터링
4. 비즈니스 요구사항에 따른 우선순위 조정
5. 데이터 품질 검증을 위한 메타 정보 포함 필수

=== 데이터 타입 권장 ===

- title: String (VARCHAR 500)
- url: String (VARCHAR 1000, UNIQUE)
- price: String/Decimal (가격 형식 다양)
- rating: Float (0.0-5.0)
- review_count: Integer
- location_tags: JSON Array 또는 Comma-separated String
- crawl_date: DateTime
- city_name: String (VARCHAR 50)
- description: Text (길이 제한 없음)
